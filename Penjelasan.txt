TAHAP 1 — SETUP
Pertama kali sistem digunakan, Admin mengisi Sheet Setup.
Di sini terdapat data referensi seperti:
Dashboard Name
Status Toko
Jenis Kunjungan
Order Ke
Kategori Sampling
Task Categories
Kompetitor
Data ini tidak menghasilkan transaksi.
Fungsinya hanya sebagai sumber dropdown untuk sheet lain.
Setup
 ↓
Menjadi sumber dropdown


TAHAP 2 — DAFTAR TOKO
Setelah Setup selesai, user mulai mengisi Daftar Toko.
Contoh:
Contact ID
Nama Toko
CRT001
Toko Maju
CRT002
Toko Jaya

Di sinilah seluruh database toko disimpan.
Field penting:
Contact ID
Nama Toko
Contact Person
Phone
Location
Status
PIC
Pada tahap ini belum ada kunjungan.
Belum ada aktivitas.
Belum ada laporan kompetitor.
Hanya pendataan toko.
Setup
 ↓
Daftar Toko


TAHAP 3 — VISIT PLAN
Setelah toko terdaftar, user membuat rencana kunjungan di Sheet Visit Plan.
Contoh:
Nama Toko
Plan Date
Toko Maju
10 Juni
Toko Jaya
11 Juni

Artinya:
"Saya berencana mengunjungi toko tersebut."
Belum benar-benar datang.
Masih berupa jadwal.
Status awal biasanya:
Scheduled

Flow:
Daftar Toko
 ↓
Visit Plan

Karena toko yang dipilih di Visit Plan berasal dari Daftar Toko.

TAHAP 4 — HARI KUNJUNGAN
Ketika tanggal kunjungan tiba.
Petugas datang ke toko.
Namun yang menarik dari file Anda adalah:
Visit Plan tidak menyimpan detail aktivitas kunjungan.
Visit Plan hanya menyimpan:
Rencana
Jadwal
Status
Sedangkan detail pekerjaan disimpan di Sheet lain yaitu:
Aktifitas


TAHAP 5 — AKTIFITAS
Inilah inti operasionalnya.
Saat kunjungan selesai, user mengisi Sheet Aktifitas.
Misalnya:
Toko : Toko Maju
Tanggal : 10 Juni
Aktifitas : Product Education
Kategori Sampling : Coffee
Keluhan : Harga terlalu mahal

Di sinilah bukti kunjungan dicatat.

Perhatikan hubungan ini:
Daftar Toko
      ↓
Visit Plan
      ↓
Aktifitas

Artinya:
Toko dibuat.
Dijadwalkan kunjungan.
Kunjungan dilakukan.
Aktivitas dicatat.

TAHAP 6 — UPDATE LAST VISIT
Ini bagian yang otomatis.
Pada Sheet Daftar Toko terdapat kolom:
Last Visit

Kolom ini tidak diisi manual.
Formula mengambil data dari:
Aktifitas

Jadi ketika user membuat aktivitas baru:
Aktifitas
↓
Tanggal Kunjungan Baru
↓
Last Visit berubah otomatis

Contoh:
Sebelumnya:
Last Visit
05 Juni

Setelah aktivitas baru:
Last Visit
10 Juni


TAHAP 7 — INTELEJEN REPORT
Saat sedang berada di toko.
User mungkin melihat aktivitas kompetitor.
Contoh:
Kompetitor:
ABC

Aktivitas:
Diskon 20%

atau
Kompetitor:
XYZ

Aktivitas:
Display baru

Informasi ini tidak masuk ke Aktifitas.
Tetapi masuk ke Sheet:
Intelegence Report

Jadi sebenarnya Intelegence Report adalah cabang dari kunjungan toko.
Flow-nya:
Saat Visit
     ├─ Aktifitas
     └─ Intelegence Report

Karena keduanya terjadi saat kunjungan yang sama.

TAHAP 8 — DASHBOARD
Dashboard tidak menghasilkan data.
Dashboard hanya membaca data dari semua sheet.
Dashboard mengambil data dari:
Daftar Toko
Untuk:
Total Toko
Status Toko

Visit Plan
Untuk:
Target Kunjungan
Scheduled
Overdue
Visited

Aktifitas
Untuk:
Aktivitas terbaru
Sampling
Feedback

Intelegence Report
Untuk:
Aktivitas Kompetitor
Monitoring Kompetitor

Flow Dashboard:
Daftar Toko
      │
Visit Plan
      │
Aktifitas
      │
Intelegence Report
      │
      ▼
Dashboard

Kalau nanti ini dijadikan CRM Web, sebenarnya user sehari-hari hanya akan bekerja di 3 menu utama:
Daftar Toko → membuat dan mengelola toko.
Visit Plan → membuat jadwal kunjungan.
Aktifitas + Intelegence Report → mengisi hasil kunjungan.
Sedangkan Setup dan Dashboard lebih banyak digunakan oleh Admin dan Manager.
